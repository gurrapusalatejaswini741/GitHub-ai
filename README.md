## ✨ What Is GitInsight AI?

**GitInsight AI** is a hackathon-grade, production-ready GitHub portfolio analyzer that uses **Claude AI (Anthropic)** and the **GitHub REST API** to give developers deep, actionable insights about their coding life — all through a visually stunning, futuristic dark dashboard.

Unlike basic GitHub stat trackers, GitInsight AI goes far beyond raw numbers. It uses large language models to:

- Determine your **developer personality archetype**
- Predict your **placement readiness** for jobs and internships
- Score your **hackathon success potential**
- Map your **skill gaps** and recommend **certifications**
- Generate a **professional portfolio summary** and **LinkedIn headline**
- Build a **personalized career roadmap**
- Let you **battle another developer** head-to-head

Whether you're a student looking to crack campus placements, an open-source enthusiast, or a seasoned engineer wanting a 360° view of your GitHub journey — GitInsight AI is built for you.

## 📊 Data & Stats Overview

Here is a full breakdown of every data point GitInsight AI collects and displays:

### From GitHub REST API
```
User Profile
  ├── login (username)
  ├── name
  ├── avatar_url
  ├── bio
  ├── company
  ├── location
  ├── blog (personal website)
  ├── email
  ├── public_repos (total repo count)
  ├── public_gists
  ├── followers
  ├── following
  └── created_at (account age)

Repositories (up to 100, sorted by updated_at)
  ├── name
  ├── description
  ├── language (primary)
  ├── stargazers_count
  ├── forks_count
  ├── watchers_count
  ├── size (KB)
  ├── open_issues_count
  ├── topics
  ├── created_at / updated_at / pushed_at
  └── license

Public Events (up to 100)
  ├── PushEvent       → commit counts, timestamps
  ├── PullRequestEvent → open-source contributions
  ├── IssuesEvent     → issue activity
  ├── WatchEvent      → starred repos
  ├── ForkEvent       → fork activity
  └── CreateEvent     → new repos/branches
```

### Computed Metrics
```
totalStars          = sum of stargazers_count across all repos
totalForks          = sum of forks_count across all repos
totalWatchers       = sum of watchers_count across all repos
avgStars            = totalStars / repo count
langMap             = { language: repo_count } dictionary
langBytes           = { language: cumulative_size_KB } dictionary
commitDays          = { "YYYY-MM-DD": commit_count } from PushEvents
commitHours         = Array[24] of hourly event counts
recentActivity      = last 30 days of commitDays entries
readmeScore         = (repos with description / total repos) × 100
openSourceContribs  = count of PullRequestEvents
consistency         = unique days with any commit in last 100 events
eventTypes          = grouped count of each event type (for activity breakdown)
```

### AI-Generated Scores (0–100 scale)
```
placementScore      → Jobs / internship readiness
hackathonScore      → Hackathon performance prediction
innovationScore     → Originality and creativity in projects
collaborationScore  → Open-source & team contribution level
projectComplexity   → Sophistication of technical work

Radar Axes (0–100 each):
  problemSolving    → Complexity and variety of solved problems
  codeQuality       → README score, project structure signals
  collaboration     → PR count, follower ratio, fork activity
  innovation        → Original repos, topic diversity
  consistency       → Daily commit patterns, streak behavior
  breadth           → Number of distinct languages used
```

## 🔑 API Configuration

### Anthropic (Claude AI)

GitInsight AI uses **Claude Sonnet 4** (`claude-sonnet-4-20250514`) for all AI analysis.

```javascript
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    // Add your key here when running outside Claude.ai:
    // "x-api-key": process.env.VITE_ANTHROPIC_API_KEY,
    // "anthropic-dangerous-direct-browser-access": "true",
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1000,
    messages: [{ role: "user", content: yourPrompt }],
  }),
});
```

> **Note:** The `anthropic-dangerous-direct-browser-access: true` header is required when calling the API directly from a browser. In production, always proxy through a backend to keep your API key secure.

### GitHub API

No API key required for public data. The app uses unauthenticated REST calls:

```javascript
const BASE = "https://api.github.com";

fetch(`${BASE}/users/${username}`)
fetch(`${BASE}/users/${username}/repos?per_page=100&sort=updated`)
fetch(`${BASE}/users/${username}/events/public?per_page=100`)
```

**Rate Limits (unauthenticated):** 60 requests/hour per IP. To increase to 5,000/hour:

```bash
# Create a GitHub Personal Access Token (read:user scope only)
# Then add to requests:
headers: {
  "Authorization": `Bearer ${GITHUB_TOKEN}`
}
```

## 🧩 Feature Breakdown

### Developer Personality Types

GitInsight AI classifies every developer into one of five archetypes based on their GitHub activity pattern, language diversity, contribution style, and repository characteristics:

| Archetype | Icon | Traits | Trigger Signals |
|-----------|------|--------|----------------|
| **Architect** | 🏗️ | Designs scalable, structured systems | Low repo count, high complexity, TypeScript/Java |
| **Hacker** | ⚡ | Fast builder, speed over perfection | High event frequency, many repos, short bursts |
| **Craftsman** | 🔨 | Writes clean, well-documented code | High README score, detailed descriptions, consistent commits |
| **Explorer** | 🚀 | Dabbles in many languages and domains | Wide language spread, varied topics, experimentation |
| **Collaborator** | 🤝 | Open-source champion, community builder | High PR count, many forks, follower-to-following ratio |

---

### Placement Readiness Score (0–100)

The AI evaluates these signals to compute placement readiness:

```
+ Diverse language stack (multi-language = industry-ready)
+ High star count (social proof from the developer community)
+ Active contribution to open source (real-world project experience)
+ Consistent commit history (discipline and work ethic)
+ Good README / documentation practices (communication skills)
+ Account longevity (sustained involvement)
+ Follower count (reputation and networking)
- Very few public repositories (limited portfolio)
- No descriptions on repos (poor documentation habits)
- Single language focus (limited versatility)
```

---

### Hackathon Success Predictor (0–100)

```
+ High commit frequency (can ship fast under pressure)
+ Language breadth (can adapt to any tech stack)
+ PushEvent density (active builder)
+ Many repositories (prototype-and-iterate mindset)
+ Recent account activity (currently active in the ecosystem)
- Low follower/star ratio (limited visibility)
- Sparse commit patterns (irregular coding habit)
```

### Battle Mode Scoring Algorithm

The winner is decided by a weighted composite score:

```javascript
function scoreUser(profile) {
  return (
    profile.totalStars      * 3.0 +   // stars are earned
    profile.repos.length    * 2.0 +   // repo count = productivity
    profile.user.followers  * 1.5 +   // community recognition
    profile.openSourceContribs * 5.0 + // PRs are high-value signal
    profile.consistency     * 2.0 +   // active days = dedication
    profile.totalForks      * 1.0     // forks = others build on your work
  );
}
```

## 📈 Scoring Algorithms

### README Quality Score

```javascript
const hasReadmeRepos = repos.filter(r => r.description).length;
const readmeScore = repos.length
  ? Math.round((hasReadmeRepos / repos.length) * 100)
  : 0;
```

### Commit Consistency

```javascript
// Counts unique days with at least one commit
// from the last 100 public events
const consistency = Object.keys(commitDays).length;
```

### Language Dominance

```javascript
// Repos using each language / total repos with a language
const langPercentage = (langCount / totalReposWithLang) * 100;
```

### Battle Score Computation

```javascript
const p1Score = totalStars * 3 + repos.length * 2 + followers * 1.5
              + openSourceContribs * 5 + consistency * 2 + totalForks;
```

> ⚠️ **Security Warning:** Never expose your `VITE_ANTHROPIC_API_KEY` in client-side code in production. For a production deployment, proxy all Claude API calls through a backend (Node/Express, FastAPI, or a serverless function) and never include the key in the browser bundle.

### Backend Proxy Setup (Production-Safe)

```javascript
// server.js (Express)
app.post('/api/analyze', async (req, res) => {
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'x-api-key': process.env.ANTHROPIC_API_KEY,  // server-side only
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(req.body),
  });
  res.json(await response.json());
});
```
